What is HTML?

HTML stands for "HyperText Markup Language", which is actually a standard for how documents are designed to be displayed on a webpage. Depending on who you're talking to, many would say that HTML is not an actual coding language, but just a markup language as the name implies. At the end of the day, HTML is the building block behind every webpage and web application!

HTML uses elements, which essentially is node that, when put together, structures a webpage. Below are elements that are present in every standard HTML file:

<html></html>: The root of an HTML document
<head></head>: Contains information about the webpage
<body></body>: Contains the contents of the webpage

Why use HTML?

In order to develop a webpage, one must abide by the standards of HTML. HTML is the leading langauge used to structure both webpages and web-based applications, so you must know HTML if you want to write even the simplest of web documents!

Below are a couple of uses for HTML:

  • To become a frontend developer, you must know HTML.
  • If you want to display even a basic document on a web browser, you must know HTML!
  • Understanding HTML provides you a better understanding of the IOT (Internet of Things).

Good practices in HTML

There are a lot of things to keep in mind when writing HTML. It's easy to get started and just throwing a bunch of elements together, but there's much more to a webpage than what a user sees. The underlying HTML must be written in a clean manner where fellow developers can follow behind you and understand the content flow.

Following the right practices and developing good habits is key to developing good HTML. But exactly is "good HTML"?

  1. Good HTML does not overuse the <div> element.
  2. Good HTML uses semantic tags, like <main> and <section> elements.
  3. Good HTML includes the proper <meta> tags contained in the <head> element.

The above rules are just a few out of many that every front-end developer should follow at a basic level. More specifically, the rules above will allow your webpage to become more accessible to users with disabilities, as well as increase the webpage's SEO (Search Engine Optimization).

Now that we explored the HTML basics, the next logical step to take is to style our webpage. But how is that done?

What is CSS?

To style modern day webpages, a CSS (Cascading Style Sheets) file is made and linked to the applicable index file. In fact, just about every single webpage you view is styled thanks to CSS. But what is CSS exactly?

CSS is a language used to do a number of things. For starters, elements are selected in the CSS language so that the developer may apply visual styles to said elements. Not only that, but CSS also can determine how a webpage or application looks on screen, paper (when a page is printed), or on other medias. Below is an example of how CSS looks:

body { background-color: black; color: white; }
header { font-size: 1.5rem; margin: 1.5rem; }

The above CSS code is creating a rule to make the <body> element of the HTML document a black background with white text, while making changing the font size of elements inside of <header> elements. The question is, how is the browser knowing which elements are being selected to apply a styling rule?

Selectors in CSS

In CSS, a selector is used to target specific elements for styling. There are many types of selectors, which can be referenced here on the w3schools website.

Below are a couple of common CSS selectors being applied:

The "type" selector:
div { width: 80%; margin: auto; }
p { font-size: 1.15rem; color: red; }

Type selectors are used to target a general element for high-level styling. These can be used to easily provide typography stylings to all of the <p> elements, or give all of the <div> elements a width and margins.

In the above example, the element types are p and div elements.

The "class" selector:
.container { width: 100%; min-height: 200px; }

Class selectors are used to target a collected group of elements that share the selected class attribute's value. To target a class attribute, a period (.) is placed before the name of the class.

In the above example, the class attribute value is "container".

The "id" selector:
#login-form { padding: 30px; min-height: 450px; }

Id selectors are used to target a single element with the specified id attribute's value. These are meant to target only one element, as it is recommended to only use one id value in an entire HTML document.

In the above example, the id attribute value is "login-form".

Good practices in CSS

Understanding proper CSS and the rules of CSS is crucial to becoming a strong frontend developer. Every webpage has to be styled for it to appeal to users. Otherwise, your application may not be used often, and you may not be taken seriously as a developer.

That being said, mastering CSS is just like mastering any other language; it takes a lot of time and practice to understand CSS works. Styling a webpage is arguably one of the most difficult parts of frontend development, and can be frustrating at times.

Here are a couple of good practices to reduce the frustration when styling webpages:

  • Use class selectors often. This will help outline your HTML, as well as target groups of elements for easier mass style changes.
  • Do not over-complicate CSS selectors. Selectors can go into depth, which will cause you and fellow developers headache if styles need to change.
  • Write comments in your CSS file. Comments look like /* content */, which can provide you and fellow developers helpful notes in massive CSS files.
  • Always keep specificity in mind. This is a very important concept. You can reference specificity here.
